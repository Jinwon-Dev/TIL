# HTTP 상태 코드

</br>

## HTTP 상태 코드란?
- **클라이언트가 보낸 HTTP 요청에 대한 서버의 응답 코드**이다.
- 상태 코드에 따라 **요청의 성공/실패 여부**를 판단한다.
- 첫 번째 숫자에 따라 크게 5가지로 분류 된다.
    - `1XX(Informational)` : 요청을 받았으며, 프로세스를 계속 진행한다.
    - `2XX(Success)` : 요청을 성공적으로 받았으며 인식했고, 수용하였다.
    - `3XX(Redirection)` : 요청 완료를 위해 추가 작업 조치(redirection)가 필요하다.
    - `4XX(Client Error)` : 요청의 문법이 잘못되었거나, 요청을 처리할 수 없다.
    - `5XX(Server Error)` : 정상적인 클라이언트의 요청에 대해 서버의 문제로 인해 응답할 수 없다.

</br>

## HTTP 상태 코드 종류와 의미

### `100번대 코드 - Information Responses`
- **서버가 요청을 받았으며 클라이언트는 작업을 계속 진행하라는 의미**로, 거의 사용되는 경우가 없으므로 생략한다.

</br>

### `200번대 코드 - Successful Responses`
- **클라이언트의 요청이 성공적으로 수행되었다는 의미**이다.
    - `200 OK` : 성공적으로 처리했을 때 쓰이고, 성공의 의미는 HTTP 메소드에 따라 달라진다.
    - `201 Created` : 요청이 성공적이며, 그 결과로 새로운 리소스가 생성되었다는 의미이다.
        - 주로 `POST` 요청 또는, 일부 `PUT` 요청 이후에 따라온다.
    - `202 Accepted` : 요청이 받아들여졌지만, 처리되지 않았음을 의미한다.
    - `204 No Content` : 성공적으로 처리했지만, 요청에 대해 보내줄 수 있는 콘텐츠가 없다는 의미이다.
    - `206 Partial Content` : 컨텐츠의 일부 부분만 제공한다는 의미이다.
        - 주로 클라이언트에서 시작 범위나 다운로드할 범위를 지정한 경우, 자동으로 해당 부분만 제공할 때 사용된다.

</br>

### `300번대 코드 - Redirection Messages`
- **이 요청을 완료하기 위해서는 redirection이 이루어져야 한다는 의미**이다.
    - `301 Moved Permanently` : 영구적으로 컨텐츠가 이동했을 때 사용된다.
        - 요청한 리소스의 URI가 변경되었음을 의미한다.
    - `302 Found` : 일시적으로 컨텐츠가 이동했을 때 사용된다.
    - `303 See Other` : 서버가 사용자의 `GET` 요청을 처리하여, 다른 URL에서 요청된 정보를 가져올 수 있도록 응답한다.
    - `304 Not Modified` : 캐시를 목적으로 사용되며, 클라이언트에게 응답이 수정되지 않았음을 알려준다.
        - 200 다음으로 많이 볼 수 있다.
    - `307 Temporary Redirect` : 302와 동일하게 일시적인 컨텐츠 이동을 의미하지만, 요청한 HTTP 메소드의 변경을 허용하지 않는다.
        - ex) `POST` -> `POST` = O, `POST` -> `GET` = X
    - `308 Permanent Redirect` : 301와 동일하게 영구적인 컨텐츠 이동을 의미하지만, 요청한 HTTP 메소드의 변경을 허용하지 않는다.

</br>

### `400번대 코드 - Client Error Responses`
- **클라이언트 측의 요청은 올바르지 않다는 의미**로, 브라우저에 직접 표출된다.
    - `400 Bad Request` : 요청 자체가 잘못되었을 때 사용하는 코드이다.
    - `401 Unauthrized` : 인증이 필요한 리소스에 인증 없이 접근할 경우 발생하는 코드이다.
        - 이 코드를 사용할 때는 반드시 브라우저에 어느 인증 방식을 사용할 것인지 보내야 한다.
        - 단순히 권한이 없는 경우, `403 Forbidden`을 사용해야 한다.
    - `403 Forbidden` : 클라이언트가 콘텐츠에 접근할 권한을 갖고 있지 않을 경우에 발생한다.
        - `401 Unauthrized`와 다른 점은, 서버가 클라이언트가 누군지 알고 있다.
        - 관리자가 해당 사용자를 차단했거나, 서버에 `index.html`이 없을 때도 발생할 수 있다.
    - `404 Not Found` : 서버가 요청받은 리소스를 찾을 수 없다는 의미이다.
        - 브라우저에서는 알려지지 않는 URL을 의미하고, 가장 흔하게 볼 수 있는 코드이다.
    - `405 Method Not Allowed` : 서버에서 허용되지 않은 메소드로 요청시 사용되는 코드이다.
        - ex) 문서 조회만 가능한데, `POST`, `PUT` 또는 `DELETE` 를 전송해서 변경하려는 경우
    - `408 Request TimeOut` : 요청 중 시간이 초과되었을 때 사용되는 코드이다.

</br>

### `500번대 코드 - Server Error Responses`
- **서버가 응답할 수 없다는 의미**이다.
    - `500 Internal Server Error` : 서버에 오류가 발생해 요청을 수행할 수 없을 때 사용된다.
        - 주로 설정이나 세미콜론을 빼먹는 등의 각종 에러로 비정상 종료를 하는 경우
    - `502 Bad Gateway` : 게이트웨이가 연결된 서버로부터 잘못된 응답을 수신했을 때 사용된다.
    - `503 Service Temporarily Unavailable` : 서비스를 일시적으로 사용할 수 없을 때 사용된다.
        - 주로 웹 서버가 과부하로 다운되었을 때 볼 수 있다.
    - `504 Gateway Timeout` : 서버가 게이트웨이 역할을 하고 있으며 적시에 응답을 받을 수 없을 때 사용된다.

</br>
